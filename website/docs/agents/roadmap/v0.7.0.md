# v0.7.0 - Agent Orchestration & Workspace

Sub-agent workers, workspace organization, and agent personalities.

## Workspace Management

- [ ] Drag-and-drop agents — rearrange agent cards in the sidebar by dragging.
- [ ] Project folders — organize agents into folders and subfolders by project.
- [ ] Create agents from browser — spawn new agent sessions directly from the web UI with voice and project selection.

## Agent Personalities

- [ ] Distinct personalities — each voice gets a subtle personality embedded in its session CLAUDE.md to increase memorability and differentiation.

## Sub-Agent Workers

Each agent can spawn lightweight workers that inherit its voice and appear nested in the sidebar. Single-level hierarchy — workers can't spawn their own workers. Leaders get tools to spawn, list, kill, and message workers. Max 4 workers per leader, auto-terminate after 30 minutes idle.

- [ ] Worker spawn/terminate — leader agents can create and destroy workers.
- [ ] Worker messaging — leaders can send messages to workers and receive responses.
- [ ] Sidebar nesting — workers appear nested under their leader in the browser sidebar.
- [ ] Resource limits — max 4 workers per leader, 30-minute idle timeout.

## Other

- [ ] Public API — token auth, versioned endpoints, agent management.
- [ ] Standalone STT/TTS installs — Whisper and Kokoro from upstream with systemd files.
- [ ] Whisper model selection — choose model size from settings.
