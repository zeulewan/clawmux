# v0.5.0 — Polish & Reliability

Smaller improvements to iOS, browser, and hub stability before the big MCP overhaul in v0.6.0.

## Hub Reliability

- [ ] **Adopt orphaned sessions on startup** — Scan for existing `voice-*` tmux sessions on hub start and re-adopt them (reconnect MCP, restore tabs). Prevents orphaned sessions after hub restart.
- [ ] **Hold-aware timeout** — Exempt sessions with pending `converse()` calls from the inactivity timeout so held sessions don't get killed.
- [ ] **Streaming TTS** — Stream audio chunks as Kokoro generates them to reduce time-to-first-audio.

## iOS Polish

### Live Activity Revamp

Three distinct Live Activity designs based on input mode:

**Auto mode** — Full Live Activity with:
- Dynamic Island compact: status dot + voice name + waveform/listening icon
- Dynamic Island expanded: voice name, status, last message, mic state indicator
- Lock Screen: voice name, status, last message truncated, "Listening" or "Speaking" pill

**PTT mode** — Compact Live Activity:
- Dynamic Island compact: voice name + mic icon
- Lock Screen: "Open to Talk" button (deep links to app, starts PTT)

**Typing mode** — No Live Activity (rely on push notifications)

### STT Edit Before Send

After PTT recording, surface transcription in an editable text box before sending to Claude:

1. Hub: `preview_transcription` mode — Whisper result sent back as `user_text_preview`, Claude blocked until iOS confirms
2. iOS: show text in inline edit field with Send/Cancel
3. User edits if needed, taps Send

### Message Editing

Long-press on a sent message to edit. Hub receives `message_edit` event and appends a correction turn for Claude.

### Action Button Integration

Register AppShortcut so users can assign "Toggle Recording" to iPhone Action Button (15 Pro, 16+).

### Background Mode Improvements

- Proper audio session deactivation on interruption
- Background VAD sensitivity tuning
- Background audio cues via system sound

## Browser

- [ ] **Sidebar navigation** — Persistent sidebar with compact voice cards replaces tab-based navigation. Status dots, unread indicators, and selected state styling. (WIP from v0.4.0)
- [ ] **Seamless reconnection** — When the browser reloads or a new tab connects, the hub sends full session state (current converse phase, listening/speaking/working status) so the UI syncs immediately without stale indicators.
- [ ] **Live reload** — Hub watches `static/hub.html` for changes and sends a WebSocket message to trigger browser reload. Sessions reconnect seamlessly via the reconnection sync above.
- [ ] **Multi-client support** — Multiple browser tabs/devices can connect simultaneously and all show accurate, synchronized session state.
- [ ] **Interrupt gesture** — Tap anywhere during playback to stop agent mid-sentence (auto mode).
- [ ] **Session elapsed time** — Show time since last message in session header.
- [ ] **Copy on long-press** — Copy chat message text.

## Distributed Audio Processing

- [ ] **Configurable STT/TTS URLs** — Make Whisper and Kokoro endpoints configurable so audio processing can run on a different machine than the agent.
