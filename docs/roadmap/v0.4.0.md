# v0.4.0

Polish, feedback, and integrations.

## Activity Indicators

- [ ] **Task progress icons** — Show small icons/animations in the session tab and chat when the agent is actively working (thinking, running commands, editing files). Lets the user know the model is busy on long tasks.
- [ ] **Interim status messages** — Periodic short spoken or text updates from the agent while working on multi-step tasks ("Looking into that...", "Running the tests now..."). Prevents silence during long operations.

## Audio & Voice

- [x] **Voice interrupt** — Interrupt Claude by speaking instead of tapping the button. VAD detects speech during playback, stops audio, and starts recording immediately. Toggle: "Auto Interrupt".
- [x] **Thinking sounds** — Subtle pulsing 440Hz tone via Web Audio API while Claude is processing. Very low volume (~0.03), stops when agent responds.
- [x] **Thinking indicator in chat** — Animated pulsing dots in the chat transcript while Claude is processing. Disappears when the agent responds.
- [ ] **Streaming TTS** — Stream audio chunks as Kokoro generates them to reduce time-to-first-audio.

## Hub Reliability

- [ ] **Adopt orphaned sessions on startup** — Scan for existing `voice-*` tmux sessions on hub start and re-adopt them (reconnect MCP, restore tabs). Prevents orphaned sessions after hub restart.
- [ ] **Hold-aware timeout** — Exempt sessions with pending `converse()` calls from the inactivity timeout so held sessions don't get killed.

## Inter-Agent Communication

- [ ] **Agent-to-agent messaging** — Allow sessions to send messages to each other through the hub. An agent can address another by voice name (e.g. "Hey Sky, what did you find?") and the hub routes it to that session's Claude instance.
- [ ] **Shared context** — Agents can read a shared scratchpad or clipboard managed by the hub for passing data between sessions without voice.
- [ ] **Voice relay** — Option to have agents speak to each other audibly through the browser so the user can listen in on the conversation.

## Integrations

- [ ] **OpenClaw integration** — Connect Voice Chat Hub to OpenClaw for iOS/mobile voice access.

## Native App

- [ ] **macOS/iOS app** — Swift app mirroring the browser client: WebSocket, audio playback, recording, conversation display.
