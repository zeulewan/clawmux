# v0.4.0

Advanced features — deeper integrations and native app.

## Audio & Voice

- [ ] **Voice activity detection** — Auto-detect when the user stops speaking and end recording. Include a toggle to disable in noisy environments.
- [ ] **Streaming TTS** — Stream audio chunks as Kokoro generates them to reduce time-to-first-audio.
- [ ] **Mid-task interjection** — Allow the user to speak while Claude is working, queuing audio for context.

## Hub Reliability

- [ ] **Adopt orphaned sessions on startup** — Scan for existing `voice-*` tmux sessions on hub start and re-adopt them (reconnect MCP, restore tabs). Prevents orphaned sessions after hub restart.

## Multi-Session UX

- [ ] **Hold/resume model** — Active tab gets full audio. Switching away pauses the session (blocks `converse()` naturally). Switching back resumes — replays buffered audio if any, then prompts for mic.
- [ ] **Background audio buffering** — When a background session's Claude speaks, buffer the TTS audio. Play it when the user switches back to that tab before resuming the conversation.
- [ ] **Notification system** — Badge + optional sound when a background session needs attention (wants to listen, or has queued speech).
- [ ] **Hold-aware timeout** — Exempt sessions with pending `converse()` calls from the inactivity timeout so held sessions don't get killed.
- [ ] **Graceful tab-switch during playback** — If user switches away mid-playback, pause audio. Resume on switch-back, then send `playback_done`.

## Native App

- [ ] **macOS/iOS app** — Swift app mirroring the browser client: WebSocket, audio playback, recording, conversation display.
