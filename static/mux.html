Why it exists

Two problems no one has solved yet

Claude Code is powerful. But using it at scale — across multiple parallel workstreams — is rough. These are the two core problems we're tackling.

🖥️
Problem 01

Remote control of Claude Code

Claude Code runs in a terminal. That's fine for one session. But when you're on your phone, on another machine, or just want to check in — there's no way to reach it. ClawMux exposes your agents through a browser and a voice interface you can access from anywhere.

🧠
Problem 02

Human context switching

Using 10 agents at once is hard. 3 is doable. You run out of mental RAM fast. The human brain didn't evolve to read terminals — it evolved to talk to people who have voices, names, and personalities. With voice, you remember who you were talking to, and you stay in flow.

"You often forget what the 6th terminal was doing. This is the first look at what agentic engineering — vibe coding — will look like in the future."
Architecture

Simple stack, powerful result

Built on tools you already know — tmux, Claude Code, and a lightweight Python hub that wires it all together.

🤖

Claude Code

Each agent is a full Claude Code session — the same tool you use in the terminal, running in a tmux pane.

⬛

tmux

Agents live in persistent tmux sessions. They survive disconnects, restarts, and context switches.

🔌

MCP Server

A custom Model Context Protocol server gives each Claude agent a voice — letting it speak and listen through the hub.

🎙️

Whisper + Kokoro

Local speech-to-text and text-to-speech. Your voice stays on your machine — nothing is sent to external services.

🐍

Python Hub

A FastAPI server manages agent sessions, routes audio, and exposes the WebSocket interface for the browser.

🌐

Browser / iOS

A clean web interface — or the native iOS app — lets you switch between agents, see chat history, and talk hands-free.
